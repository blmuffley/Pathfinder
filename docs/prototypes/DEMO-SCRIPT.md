# Avennorth Pathfinder — Demo Script

## Setup

```bash
cd docs/prototypes/workspace-app
npm run dev
# Opens on http://localhost:4200
```

**URL: http://localhost:4200**
**Duration: 12-15 minutes**

### Port Assignments (No Conflicts with Other Avennorth Products)

| Service | Port | Purpose |
|---------|------|---------|
| Workspace Prototype | **4200** | This demo |
| Gateway (gRPC) | 8443 | Production agent traffic |
| Shared AI Engine | 8080 | Claude API orchestration |
| Integration Intelligence | 8081 | Health scoring, EA recon |
| CMDB Ops Agent | 8082 | 8 autonomous agents |
| Service Map Intelligence | 8083 | Coverage, risk, impact |
| PostgreSQL | 5432 | Gateway data store |

*Avennorth Compass and other products typically use ports 3000-3100 and 5173. Pathfinder uses 4200 (prototype), 8080-8083 (intelligence), and 8443 (gateway) to avoid conflicts.*

## Demo Script

### Opening (1 minute)

> "This is Avennorth Pathfinder — our CMDB-first integration discovery and intelligence platform built for ServiceNow. What you're seeing is the Polaris workspace that integration analysts use every day. Let me walk you through what it does."

---

### Act 1: The Overview (2 minutes)

**Page: Overview** *(landing page)*

> "The overview gives you the full picture at a glance."

**Point out:**
- **KPI tiles** at top — "14 integrations discovered, 9 healthy, 3 degraded, 1 critical. 12 active agents across our fleet. 6 open coverage gaps that need remediation."
- **Health distribution donut** — "Most of our estate is green, but we have that one critical integration and three degraded ones that need attention."
- **Integration types donut** — "Pathfinder auto-classifies every integration — API, Database, Messaging, Email — based on port and process analysis. No manual tagging."
- **Health trend sparkline** — "The 30-day trend shows us availability has been stable until this week — see that dip? We'll come back to that."

**Click** the "Critical" KPI tile (red, value "1") — this navigates to the Integrations page.

---

### Act 2: Integration Discovery & AI Intelligence (4 minutes)

**Page: Integrations**

> "This is where the power is. Every row is a discovered integration — automatically found by eBPF agents watching kernel-level network traffic. No code changes, no config files."

**Demo the filters:**
1. **Click "Critical"** filter button — shows only the critical integration (Notification Service → SendGrid)
2. **Click "All"** to reset
3. **Type "kafka"** in the search box — shows only the Kafka integration
4. **Clear search**, **click "Database"** type filter — shows the 3 database integrations

> "Analysts can slice and dice instantly. Now let me show you the detail view."

**Click** the **"Notification Svc → Email Gateway"** row (Critical, score 34)

> "This is the integration that's in trouble. Look at the split view — list on the left, detail on the right."

**Overview tab** (default):
- **Point to the AI Summary** (red-bordered card) — Read it aloud:
  > "The AI is telling us: 4.8% error rate, availability dropped to 96.2%, we're likely hitting SendGrid rate limits. It's even recommending the fix — review sending volume and implement backoff. This was generated by Claude, not a human."
- **Point to Classification** — "Source, target, type, confidence 91% — all auto-discovered."
- **Point to Governance** — "Criticality is High, owner is assigned, but notice data classification is 'Internal'. For an email gateway handling customer data, that might need review."

**Click "Interfaces" tab:**
> "Two interfaces — port 25 and port 587, both SMTP. Look at the error rates: 4.8% and 3.2%. The P99 latency is 2.4 seconds on port 25. This confirms the AI's diagnosis."

**Click "Health" tab:**
> "The 30-day trend chart. See the spike in error rate in the last few days? And latency climbing? The AI saw this pattern and flagged it before anyone filed a ticket."

**Click "AI Summary Refresh" button** on Overview tab:
> "Watch — I can refresh the AI analysis on demand."
*(Toast appears: "AI summary refreshed successfully")*

**Click** the **"Event Bus → Kafka Cluster"** row (Healthy, score 96):
> "Compare that to a healthy integration. Score 96, 890,000 flows — this is the backbone event bus. The AI summary says zero message loss in 90 days across 12 partitions. That's confidence."

---

### Act 3: Agent Fleet (2 minutes)

**Click "Agent Fleet"** in sidebar

> "Every server running a Pathfinder agent shows up here. 12 active, 2 stale."

**Point out:**
- **OS distribution** — "Linux, Windows, and Kubernetes nodes. One agent binary, three deployment models."
- **Monospace agent IDs** — "Each agent gets a UUID on enrollment. Persists across restarts."
- **Stale agents** (amber rows) — "staging-web-01 hasn't sent a heartbeat in 3 days. dev-app-01, 5 days. These automatically trigger coverage gap records."

**Search** for "k8s" — shows the two Kubernetes nodes

**Click the decommission button** (power icon) on staging-web-01:
> "If we know a server is being retired, we can decommission the agent right from here."
*(Confirmation dialog appears)*
> "It warns us that a coverage gap may be created. This connects to the self-healing loop."
**Click Cancel** (don't actually decommission in the demo)

---

### Act 4: Coverage Gaps & Self-Healing (2 minutes)

**Click "Coverage Gaps"** in sidebar

> "This is the Kanban board for coverage gap remediation. Think of it as the ops team's triage view."

**Walk through the lanes:**
- **Open** (6 items) — "These servers need agents. Prioritized: Critical and High at the top."
- **In Progress** (2 items) — "Change requests have been created. Agents are being deployed."
- **Resolved** (2 items) — "Self-healing worked. Agents deployed, verified, gap closed."
- **Waived** (2 items) — "Dev test servers — we decided they don't need coverage."
- **Failed** (1 item) — "prod-mq-02 failed to deploy. The flow creates a notification for manual intervention."

**Click "Create CR"** on the Critical gap (prod-mail-01):
> "One click to create a ServiceNow change request. In production, this triggers the Flow Designer self-healing loop — it checks prerequisites, creates a standard or normal CR, deploys the agent, verifies enrollment, and closes the gap automatically."
*(Toast: "Change request created for prod-mail-01")*

**Click "Waive"** on the Low priority gap (staging-db-01):
*(Dialog appears with textarea)*
> "For servers that don't need coverage, analysts can waive with a documented reason. This feeds into compliance reporting."
**Type** "Staging database — scheduled for decommission next sprint"
**Click Waive**
*(Toast: "Gap waived for staging-db-01")*

---

### Act 5: EA Reconciliation (2 minutes)

**Click "EA Reconciliation"** in sidebar

> "This is where we bridge the gap between what Pathfinder discovers and what Enterprise Architecture has documented."

**Point to KPIs:**
> "5 unmapped integrations, 5 mapped, 1 suggested, 1 disputed. The progress bar shows we're 36% reconciled."

**Click** "Auth Service → LDAP Directory" on the left panel:
> "The AI found two potential EA matches. The first one — 85% confidence — is a fuzzy name match: 'auth service' matches 'authentication service' with edit distance 1. The second — 52% — is a group match because both CIs belong to the Identity & Access Management business service."

**Click "Confirm"** on the 85% match:
*(Toast: "EA mapping confirmed")*
> "One click. In ServiceNow, this updates the Integration CI's EA status to 'Mapped' and links it to the EA relationship record. The analyst just closed a reconciliation gap in 5 seconds that would have taken a spreadsheet exercise."

**Click** "CRM → Salesforce Sync" (Disputed):
> "This one is disputed — the EA record was deleted but the integration still exists. The AI found two candidates at 92% and 71% confidence."

---

### Act 6: Health Dashboard (2 minutes)

**Click "Health Dashboard"** in sidebar

> "The executive view. Everything we've seen at the individual integration level, aggregated."

**Toggle time ranges:**
- **Click "7d"** — shows last 7 days
- **Click "30d"** — shows full 30-day trend with the anomaly spike visible

> "See the error rate spike in the last few days? That's our SendGrid integration dragging the aggregate. And latency climbing — that's the Elasticsearch cluster we saw was degraded."

**Point to sparkline cards:**
> "Four key metrics at a glance: 99.7% availability, 72ms average latency, 0.8% error rate, 5.1k requests per second throughput. Each with a mini trend line."

**Point to "Attention Required" list:**
> "Three integrations need attention — the critical email gateway, the degraded LDAP, and the degraded Elasticsearch. Each links back to the detail view."

**Click "View all →"** — navigates back to Integrations

---

### Closing (1 minute)

**Navigate back to Overview**

> "What you've just seen is Avennorth Pathfinder doing what no other ServiceNow tool does:
>
> 1. **Zero-code discovery** — eBPF agents find every integration automatically
> 2. **AI intelligence** — Claude-powered health scoring, summarization, and remediation advice
> 3. **Autonomous CMDB governance** — 8 AI agents continuously cleaning your CMDB
> 4. **Self-healing coverage** — gaps detected, change requests created, agents deployed, verified, and closed automatically
> 5. **EA reconciliation** — AI matches discoveries to EA records in seconds, not spreadsheets
>
> All of this runs inside your ServiceNow instance as a Polaris workspace. No separate tool. No context switching. Your CMDB analysts live here."

**Pause.**

> "Questions?"

## Talking Points for Q&A

| Question | Answer |
|----------|--------|
| "How does it discover integrations?" | "eBPF probes in the Linux kernel watch TCP/UDP connections. No code changes, no SDK, no config files. <1% CPU overhead. We also support Windows ETW and Kubernetes DaemonSets." |
| "What about false positives?" | "Every integration gets a confidence score based on port rules, process names, flow count, and consistency. The confidence threshold is configurable — default 0.8 means only high-confidence integrations become CIs." |
| "How does the AI work?" | "Claude API via our Shared AI Engine. The integration context — source, target, type, interfaces, health metrics — is sent as structured prompts. Claude returns JSON with summaries, health assessments, and recommendations." |
| "What's the pricing?" | "Annual subscription by Managed Node count. Two packages: Standard (Discovery + CMDB Ops) starting at $50K/yr, Professional (adds Integration Intelligence + Service Map Intelligence) starting at $100K/yr. Pricing tiers scale by node count — S up to 500, M up to 2,000, L up to 5,000, XL custom. If our agent runs on it, it's a Managed Node. Everything else we discover is free." |
| "How long to deploy?" | "Crawl stage in 4 weeks: 10-20 servers, validate accuracy. Walk stage by week 12: full production coverage with intelligence enabled. Run stage by week 24: autonomous operations." |
| "Does it replace ServiceNow Discovery?" | "No — it complements it. ServiceNow Discovery maps infrastructure. Pathfinder discovers the integration relationships BETWEEN infrastructure and adds AI intelligence on top." |
